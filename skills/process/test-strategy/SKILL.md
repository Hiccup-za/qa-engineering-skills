# Test Strategy Definition

Define testing goals, levels, and quality gates for a repository.

## When to Use
- Starting QA for a new repo or major initiative
- Existing testing is inconsistent or unclear

## Instructions
- Identify primary user flows and business risks.
- Decide testing levels (unit, integration, end-to-end) and scope.
- Set quality goals (coverage targets, defect thresholds, SLAs).
- Define environments, data needs, and constraints.
- Document tradeoffs and assumptions.

## Inputs
- Product goals or roadmap
- Architecture overview
- Release cadence and constraints

## Outputs
- Test strategy with goals, levels, and quality gates
- Agreed scope and constraints

# Metrics and Quality Reporting

Define QA metrics and reporting to track quality, risk, and progress.

## When to Use
- You need a shared view of quality health
- Releases lack objective go/no-go signals

## Instructions
- Choose a small set of actionable metrics.
- Define measurement sources and cadence.
- Set thresholds and escalation rules.
- Create a simple reporting format.
- Review metrics regularly and adjust as needed.

## Inputs
- Test results and defect data
- Release schedule and stakeholders
- Existing tooling or reports

## Outputs
- Metrics definition and thresholds
- Lightweight quality report format

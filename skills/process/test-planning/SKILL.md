# Test Planning

Define the scope, risks, and approach for testing a change or release.

## When to Use
- New feature work, significant refactor, or release planning
- When risks or dependencies are unclear

## Instructions
- Identify scope and key user flows.
- Assess risks and prioritize test focus areas.
- Define test approach (levels, types, tooling).
- Estimate effort and align on timelines.
- Document assumptions and open questions.

## Inputs
- Requirements or user stories
- Architecture notes or designs
- Release timeline and stakeholders

## Outputs
- Test plan with scope, risks, and approach
- Agreed priorities and timelines

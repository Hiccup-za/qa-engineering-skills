# ISTQB Foundation

Apply ISTQB Foundation knowledge areas to QA work and team practices.

## When to Use
- Establishing a common QA baseline for a team
- Aligning terminology and practices across roles

## Instructions
- Use the ISTQB principles to set shared testing language.
- Apply the test process to define planning, monitoring, and control.
- Use static testing to find issues early (reviews, walkthroughs).
- Select test techniques for coverage and risk (black-box, white-box).
- Plan testing activities using management topics (risk, configuration, defect).
- Use tools to support traceability, execution, and reporting.

## Knowledge Areas
- Testing fundamentals and principles
- Test process and levels
- Static testing and reviews
- Test techniques (black-box, white-box, experience-based)
- Test management (risk, estimates, monitoring, defects)
- Tool support for testing

## Inputs
- Product or project context
- Existing QA practices and artifacts

## Outputs
- Shared QA baseline aligned to ISTQB Foundation topics
- Identified gaps and priority areas for improvement
